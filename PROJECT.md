Консольное приложение, созданное для упрощения процесса стресс-тестирования

# **[ ПРОБЛЕМНОЕ ПОЛЕ ]**

Большинство учеников кружков по программированию решают много задач, у которых есть более простые, но менее оптимальные решения. Часто более эффективный код тяжело написать правильно с первого раза, а ошибки, допущенные при этом, трудно искать, не представляя, в чём именно заключается проблема. Для решения этой проблемы был придуман метод "стресс-тестирования", но его реализация остаётся сложна для неподготовленного пользователя.
Проблема актуальна сейчас, так как много учеников обучается и будет обучаться алгоритмам и структурам данных, тем самым делая продукт востребованным.
В настоящее время основная часть аудитории не решает её никак, ограничиваясь минимальном ручным тестировании, а небольшая часть реализует стресс-тестирование, затрачивая на это много сил и времени.

# **[ ЦЕЛЬ ПРОЕКТА]**

**StressForge** - быстрое и простое стресс-тестирование

**StressForge** - инструмент, который:
- Упрощает процесс запуска стресс-тестирования;
- Позволяет пользователю быть полностью независимым и  контролировать процесс тестирования, при этом сохраняя основную сложность реализации тестирования на стороне продукта;
- Упрощает написание стресс-тестов;
- Не требует от пользователя понимания процесса тестирования.

# **[ ЦЕЛЕВАЯ АУДИТОРИЯ ]**

Для продукта можно выделить несколько социальных групп пользователей, различающихся по возрасту, уровню подготовки и целям использования инструмента.

- Основная: учащие старших классов и студенты, изучающие алгоритмы и структуры данных.
- Доп. аудитория: Составители задач.

Для данной аудитории стресс-тестирование особенно востребовано.
# **[ ЦЕННОСТЬ ПРОДУКТА ]**

- Пользователь экономит силы и время на действиях, которые выполняет продукт.
- Бизнес получает лояльного пользователя, более склонного к использованию других продуктов компании.
- Продукт уникален подходом к аудитории. "От бывших учеников нынешним".
- Преимущество перед конкурентами достигается за счёт упрощения интерфейса и малой потери функциональности.

# **[ АНАЛИЗ РЫНКА И КОНКУРЕНТОВ ]**

Платных аналогов не было найдено. Среди `open-source` проектов есть различные проекты со своими проблемами.

| Продукт                                 | Описание                                              | Ограничения                                                                                                             |
| --------------------------------------- | ----------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| [jay](https://github.com/sslotin/jay)   | Аналог продукта                                       | Не закончен, заброшен                                                                                                   |
| Judge-подобные системы                  | Платформа для тестирования решений по заданным тестам | Перегруженный интерфейс, ограниченный функционал для решения проблемы                                                   |
| [OnlineGDB](https://www.onlinegdb.com/) | Онлайн IDE, позволяющее писать код на разных языках   | Не имеет встроенного стресс-тестирования. Будучи аналогом IDE, содержит избыточный функционал и перегруженный интерфейс |

Таким образом, **StressForge** занимает уникальную нишу, предлагая простое стресс-тестирование без избыточных функций.

# **[ ОСНОВНЫЕ ФУНКЦИИ ]**

Передача корректного и некорректного решения, а также генератора тестов. Продукт запускает генератор, после чего передаёт его вывод в корректное и некорректное решение. Процесс продолжается до момента нахождения вывода генератора, при котором вывод обоих решений различны.

# **[ РЕАЛИЗАЦИЯ ]**

С помощью библиотек для работы с операционной системой запускаются  `.exe`-файлы в указанном порядке, а ввод и вывод направляются соответствующим образом. При обнаружении несовпадения программа выводит тест в консоль и сохраняет его в файл.

На создание MVP закладывается 2 месяца.

# **[ План обновлений после создания MVP ]**
1. Добавить возможность чтения заранее записанных тестов.
2. Добавить возможность пользователю передавать `checker`, который сравнивает ответы (по умолчанию побайтовое сравнение).
3. Добавить возможность пользователю создавать простой генератор в интерфейсе самой программы.
4. Реализовать поддержку запуска файлов не только в формате `.exe`, но и исходных `.cpp` и `.py`.
5. Добавить поддержку стресс-тестов для интерактивных задач.
6. Добавить возможность пользователю передавать `logger`, который выводит различные состояния элементов программ.
7. Добавление многопоточного тестирования.
8. Обновление внешнего вида (фронтенд).
